# Destiny.OS - 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트명:** Destiny.OS
- **컨셉:** 토스 생태계 기반의 사주와 MBTI를 결합한 하이브리드 운세 미니앱
- **목표:** 사용자의 MBTI와 사주 정보를 결합하여 개인화된 운세 및 성향 분석 제공

## 2. 타겟 오디언스
- MZ세대를 주축으로 한 토스 사용자
- 사주와 MBTI 등 자아 탐구 및 운세 콘텐츠에 관심이 많은 사용자

## 3. 핵심 기능 (MVP - Option 3 + AI)
> 로그상 'Option 3'의 구체적 상세 항목이 없어 일반적인 운세 앱의 고급 기능을 가정하여 작성함. (사용자 확인 필요)

### 3.1 사용자 정보 입력
- 생년월일시 (양/음력, 윤달) 입력
- 성별 입력
- MBTI 입력 (또는 약식 테스트 기능)

### 3.2 사주-MBTI 분석 엔진
- 만세력 알고리즘을 통한 사주 명식 산출
- MBTI 성향과 사주 정보의 매핑 로직
- **AI 추가 상담 기능:**
    - 분석 결과에 대해 AI에게 1회 질문할 수 있는 프롬프트 제공
    - LLM(OpenAI/Claude 등) 연동을 통한 자연어 답변

### 3.3 운세 리포트
- 오늘의 운세
- 성향 분석 리포트 (MBTI x 사주)
- (Option 3 범위에 따라 추가 될 수 있음: 궁합, 1년 운세 등)

## 4. 디자인 컨셉
- Destiny.OS라는 이름에 걸맞은 미래지향적이고 깔끔한 UI
- 토스 앱 디자인 랭귀지와의 조화 (미니앱인 경우)

## 5. 배포 전략
- **플랫폼:** iOS, Android (Flutter 기반 크로스 플랫폼)
- **배포:** 앱스토어, 플레이스토어 등록
