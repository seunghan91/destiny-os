## 앱인토스 개요

### 앱인토스란?
**앱인토스(Apps in Toss)**는 토스 앱 내에서 서드파티 서비스를 제공할 수 있는 **미니앱 플랫폼**입니다.

| 항목 | 내용 |
|------|------|
| **런칭** | 2025년 7월 (퍼블릭 베타) |
| **사용자 기반** | 토스 누적 가입자 3,000만 명 |
| **핵심 가치** | 별도 앱 설치 없이 즉시 서비스 이용 가능 |
| **아키텍처** | App-in-App (웹뷰 기반 미니앱) |

### 핵심 장점
- **사용자 확보 비용 절감**: 3,000만 토스 유저에게 즉시 노출
- **간편한 온보딩**: 토스 계정/결제수단 자동 연동
- **빠른 출시**: 평균 5일 내 프로덕션 배포 가능
- **베타 기간 수수료 면제**: 현재 서비스 수수료 무료

## 입점 자격 및 적합성 분석

### 서비스 카테고리 분류

| 카테고리 | 상태 | Destiny.OS 해당 여부 |
|----------|------|---------------------|
| 모빌리티 | ✅ 허용 | ❌ |
| 커머스/쇼핑 | ✅ 허용 | ❌ |
| **게임/엔터테인먼트** | ✅ 허용 | ⭐ **해당** |
| **라이프스타일** | ✅ 허용 | ⭐ **해당** |
| 금융 서비스 | ❌ 제한 | ❌ |
| 도박/사행성 | ❌ 금지 | ❌ |

### Destiny.OS 적합성 평가

| 평가 항목 | 상태 | 비고 |
|-----------|------|------|
| 서비스 카테고리 | ✅ 적합 | 라이프스타일/엔터테인먼트 |
| 사행성 여부 | ✅ 안전 | 운세 서비스는 도박이 아님 |
| 수익 모델 | ✅ 가능 | 인앱결제, 구독, 광고 가능 |
| 기술 스택 | ⚠️ 조정 필요 | Flutter → 웹앱 변환 필요 |
| 사업자 등록 | ⚠️ 확인 필요 | 사업자등록증 필수 |

### ⚠️ 주의 사항
- **민감 콘텐츠 가이드라인**: 운세/사주 서비스는 "미신조장" 관련 심사 가능성
- **광고 문구 심사**: "정확도 100%", "반드시 맞는다" 등 과장 표현 금지
- **AI 상담 고지**: AI 기반 상담임을 명확히 고지 필요

## 입점 절차

### 단계별 프로세스

```
1. 입점 신청 (1-2주)
   └── toss.im/apps-in-toss 에서 파트너 신청
   └── 서비스 개요, 사업자 정보 제출

2. 사전 심사 (1-2주)
   └── 서비스 적합성 검토
   └── 필요시 추가 서류 요청

3. 개발자센터 접근 권한
   └── developers-apps-in-toss.toss.im 접속
   └── 앱 프로젝트 생성 및 설정

4. 개발 및 연동 (2-4주)
   └── 토스 디자인 시스템 적용
   └── 토스 SDK/API 연동
   └── 인증, 결제 시스템 통합

5. 심사 (1-2주)
   └── 운영 심사: 약관, 정책 준수
   └── 기능 심사: 정상 동작 확인
   └── 디자인 심사: UI/UX 가이드라인
   └── 보안 심사: 취약점 점검

6. 출시
   └── 토스 앱 내 노출 시작
```

### 예상 소요 기간
- **총 소요**: 약 2-3개월
- **개발 공수**: 약 4-6주 (Flutter → 웹앱 마이그레이션 포함)

### Destiny.OS 기술 전환 필요사항

현재 Destiny.OS는 **Flutter 네이티브 앱**이므로 앱인토스 입점을 위해 **웹앱 변환**이 필요합니다.

#### 옵션 1: Flutter Web 활용 (권장)
```
장점:
- 기존 코드베이스 최대 활용
- Dart 로직 재사용
- 개발 기간 단축

단점:
- 번들 사이즈 최적화 필요
- 웹 특화 UX 조정 필요
```

#### 옵션 2: React/Next.js 신규 개발
```
장점:
- 토스 디자인 시스템과 자연스러운 통합
- 최적화된 웹 성능
- 토스 예시 코드 활용 가능

단점:
- 개발 기간 증가
- 사주 계산 로직 재구현 필요
```

### 현재 Destiny.OS 기술 스택 호환성

| 기술 | 현재 | 앱인토스 호환 |
|------|------|--------------|
| Flutter | ✅ 사용중 | ⚠️ Flutter Web 변환 필요 |
| Supabase | ✅ 사용중 | ✅ 그대로 사용 가능 |
| Firebase | ✅ 사용중 | ⚠️ FCM → 토스 알림 전환 검토 |

| BizRouter AI | ✅ 사용중 | ✅ 그대로 사용 가능 |

## Destiny.OS 적용 방안

### 1단계: 웹앱 버전 개발 (Flutter Web)

```bash
# Flutter Web 빌드 설정
flutter build web --release --web-renderer canvaskit
```

**주요 작업:**
- `web/` 폴더 최적화
- 모바일 퍼스트 반응형 UI 조정
- 토스 디자인 시스템 TDS 완전 적용
- 로딩 최적화 (Lazy loading, Code splitting)

### 2단계: 토스 SDK 연동

```dart
// 토스 인증 연동 예시 (개념)
class TossAuthService {
  Future<TossUser> loginWithToss() async {
    // 토스 SDK 호출
    final authResult = await TossAuth.login();
    
    // Supabase 계정과 연동
    await supabase.auth.signInWithCustomToken(
      token: authResult.accessToken,
    );
    
    return authResult.user;
  }
}
```

### 3단계: 결제 시스템 연동

**수익화 모델 제안:**

| 상품 | 가격 | 설명 |
|------|------|------|
| 프리미엄 사주 분석 | ₩3,900 | 상세 사주 해석 리포트 |
| AI 상담 10회권 | ₩9,900 | GPT-4o 기반 맞춤 상담 |
| 월간 구독 | ₩4,900/월 | 무제한 기능 + 일일 운세 |
| 궁합 분석 | ₩2,900 | 커플 궁합 상세 리포트 |

### 4단계: 심사 대비

**체크리스트:**
- [ ] 개인정보처리방침 작성
- [ ] 이용약관 작성
- [ ] AI 상담 면책 고지 추가
- [ ] 운세 서비스 엔터테인먼트 목적 고지
